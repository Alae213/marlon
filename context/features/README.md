This index helps readers quickly navigate the feature docs in `context/features/`.

- `context/features/Order_Management/` - Order operations docs covering the live orders surface, status flow, details, notes, filters, and dispatch from Orders.
- `context/features/Store_Workspace/` - Merchant workspace docs for store creation, dashboard entry, catalog editing, content editing, preview, and navigation.
- `context/features/Public/` - Public storefront docs for catalog browsing, product detail, cart behavior, checkout, and storefront content rendering.
- `context/features/Settings/` - Merchant settings docs for store info, billing and unlocks, delivery providers, and delivery pricing.
- `context/features/Platform/` - Cross-cutting platform docs for authentication, tenant access, billing locks, subscriptions, payments, webhooks, and runtime alignment planning.
- `context/features/Delivery_Internal/` - Internal delivery system docs for provider adapters, credentials, rollout gates, recommendations, and analytics rollups.
- `context/features/Scaffolded/` - Placeholder or deferred feature docs for surfaced but not fully built areas such as marketing, agency mode, and other scaffolds.

`context/features/_template.md` is the base template for feature docs.

These docs are truth-first and may label areas as `Current`, `Partial`, `Planned`, or `Policy-locked`.
