# Tech Stack

> Generated by setup wizard. Updated from codebase analysis.

## Frontend

- **Framework**: Next.js 16.2.0 (App Router)
- **Styling**: Tailwind CSS 4 + Custom CSS design tokens
- **Component Library**: Radix UI primitives + shadcn patterns (custom components)
- **Animation**: Framer Motion 12 + Motion library (for React 19)
- **Icons**: Lucide React
- **State Management**: React Context (cart, billing, realtime, toast)

## Backend

- **Runtime**: Convex (backend-as-a-service, serverless)
- **API Style**: Convex queries and mutations
- **Database**: Convex built-in NoSQL database
- **Real-time**: Convex subscriptions (< 100ms updates)

## Auth

- **Provider**: Clerk (@clerk/nextjs)
- **Method**: Google OAuth only (no email/password)
- **Integration**: @convex-dev/auth for Convex auth bridging
- **User Data**: clerkId, email, name, profileImageUrl, createdAt

## Payments

- **Provider**: Abstracted (configurable via PAYMENT_PROVIDER env var)
- **Current**: Chargily Pay (placeholder for future providers)
- **Supported**: EDAHABIA, CIB (SATIM), bank transfer
- **Configuration**: 
  - Set `PAYMENT_PROVIDER=chargily|sofizpay|custom`
  - API keys in environment variables
- **Webhook**: `/api/chargily/webhook` (handles payment success → activates store)

## Infrastructure

- **Frontend Hosting**: Vercel
- **Backend Hosting**: Convex Cloud
- **Analytics**: Vercel Analytics
- **Performance Monitoring**: Vercel Speed Insights

## Rationale

1. **Next.js + Convex**: Fast development with real-time updates out-of-the-box. Perfect for multi-tenant SaaS with live order updates.

2. **Clerk**: Simple Google OAuth for non-technical merchants. No password management needed.

3. **Convex Backend**: Eliminates need for separate API server. Real-time subscriptions are critical for order management.

4. **Chargily Pay**: Algeria's most accessible payment gateway. Supports EDAHABIA and CIB cards.

5. **ZR Express + Yalidine**: Two most popular delivery services in Algeria. Merchants can choose based on their region.

6. **Abstracted Payments**: Allows switching payment providers without code changes (PAYMENT_PROVIDER env var).
